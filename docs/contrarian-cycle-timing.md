# 反向周期择时策略

`volume_contraction_timing` 是默认个股策略。它不再按“趋势越强、分数越高、越值得买”做决策，而是先用最近最多 240 个已完成交易日识别周期阶段，再输出一个与阶段匹配的动作。

## 周期阶段

| 阶段 | 默认动作 | 含义 |
| --- | --- | --- |
| `declining` | `watch` / `low_buy` | 仍在下跌；只有位置不高、缩量、跌速未加快且有独立安全证据才低吸 |
| `declining_exhaustion` | `accumulate` | 价格与量能均在极低分位，且有衰竭、底背离、波动收敛或承接改善 |
| `range_bound` | `watch` | 横盘震荡，等待箱体突破或破位后重新评估 |
| `advancing` | `hold` | 上涨结构健康，高位也不会因趋势强而输出买入 |
| `advancing_weakening` | `reduce` | 高位上涨至少两类独立证据转弱，且至少一类来自非成交量指标 |
| `advancing_exhaustion` | `exit` | 极高位出现至少两类独立衰竭证据，不必等待明显转头 |
| `high_level_breakdown` | `reduce` / `exit` | 长期仍处相对高位，但已经出现放量、动能、波动和结构等多维破位 |
| `structural_risk` | `exit` | 退市风险、财务造假、审计否定、债务违约或核心逻辑反转等硬风险 |
| `unknown` | `watch` | 有效历史数据不足 60 个交易日 |

## 机器证据

周期分析器综合120/240日价格分位、多周期收益与回撤、均线斜率、成交量分位与量比、上下跌日量能、RSI/MACD 背离、ATR、布林带宽度、趋势效率和支撑压力。30%/70%是强位置证据而非绝对开关：30%-45%偏低软区和55%-70%偏高软区需要比强位置多一项独立确认。

历史分位只使用各市场最后已完成的日线。盘中实时价格可作补充，但未收盘成交量不会混入 5/20 日量比或历史分位。

## 服务端约束

- 模型用基本面、行业、新闻、资金与筹码解释和确认机器阶段。
- 当价格位置、量能和涨跌速度等机器前置条件已经成立时，模型提供的资金筹码、行业市场或基本面方向性证据可以补足最后一个独立确认维度；这些确认记录在 `external_confirmation_dimensions`，不能绕过机器前置条件。
- 模型输出与阶段冲突时，服务端会校正信号并在理由中记录。
- 高位或上涨阶段不允许低吸/抢筹；加速下跌或放量破位不允许低吸/抢筹。
- 上涨缩量或价格极高的单一证据不能卖出。
- “若出现”“可能”“未发现”“未确认”“需核验”等文本只作为观察风险，不得被服务端当成已确认硬风险清仓。
- 资金流缺失只降低置信度，不自动改变已成立的主信号。
- `sentiment_score` 仅用于旧接口兼容，由最终信号反向生成，不参与决策。

## 报告输出

飞书文字报告按清仓、减仓、抢筹、低吸、观察、持有的风险优先顺序排列，隐藏兼容分数。低吸/抢筹展示介入观察区与失效位，减仓/清仓展示压力或退出区，横盘展示箱体，持有展示防守位。完整 Markdown 报告仍保留技术面、基本面和新闻细节。
